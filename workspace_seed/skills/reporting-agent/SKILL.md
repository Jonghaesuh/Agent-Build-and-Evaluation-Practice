# Reporting Agent

`data-model-agent` skill의 산출물(모델 코드, 전처리 코드, 데이터)을 입력으로 받아
성능 평가 → 결과 분석 → 보고서(PPT/Markdown) 생성까지 수행한다.

## 언제 사용하는가
- 모델 개발/코드 변경이 완료되어 성능 평가가 필요할 때
- 데이터가 변경되어 기존 모델을 재평가해야 할 때
- 고객 공유용 보고서 또는 내부 검증 문서가 필요할 때

## 전체 흐름

```
평가 대상 확정 (모델 버전 + 데이터 버전 참조)
  → [1] 성능 평가 실행 (지표 계산)
  → [2] 결과 분석 및 개선 포인트 요약
  → [3] PPT(고객向) / Markdown(내부용) 보고서 생성
  → Checkpoint (보고서 최종 승인) — 승인 전 고객에게 전달 금지
```

---

## [1] 성능 평가 실행

1. 평가 대상 모델/코드/데이터의 **버전을 명시적으로 확인**한다. 버전이 불명확하면
   임의로 최신본을 가정하지 말고 사용자에게 확인을 요청한다.
2. 문제 유형(분류/회귀/랭킹 등)에 맞는 표준 지표를 **최소 2개 이상** 선택하고,
   왜 그 지표를 선택했는지 근거를 함께 제시한다. 이때 단순히 "표준 지표라서"가 아니라
   **데이터/문제 특성(클래스 불균형 여부, 이상치 민감도, 비즈니스 임계값 존재 여부 등)과
   연결**해 근거를 작성한다.
   - 분류: Accuracy, Precision/Recall, F1, AUROC 등 (클래스 불균형 시 Accuracy 단독 사용 지양)
   - 회귀: RMSE, MAE, R² 등 (이상치 민감도에 따라 MAE vs RMSE 선택)
   - 랭킹/추천: NDCG, MAP 등
3. 이전 버전 또는 baseline 대비 성능 변화를 **수치 + 증감률**로 반드시 함께 제시한다.
4. 평가 코드는 sandbox에서 실제 실행하여 얻은 값만 사용한다. 추정값을 채워넣지 않는다.

## [2] 결과 분석 및 개선 포인트

1. 성능 저하/개선의 원인을 제시할 경우 **"가설"임을 명시**하고, 검증되지 않은 주장을
   단정적으로 서술하지 않는다.
   - 예: "~일 가능성이 있음(가설, 추가 검증 필요)"
2. 개선 포인트는 실행 가능한 액션 아이템으로 정리한다.
   - 예: "클래스 B 데이터 추가 수집 필요", "피처 X 엔지니어링 검토"
3. 인사이트 초안은 사람이 비즈니스 맥락으로 최종 해석/보완한다는 점을 전제로 작성한다.

## [3] 보고서 / 슬라이드 생성

- **고객向 PPT**: 요약 중심. 핵심 지표, baseline 대비 변화, 주요 개선 포인트 위주로
  구성하고 세부 구현 내용은 생략한다.
- **내부용 Markdown**: 재현 가능한 상세 근거 포함 — 모델/데이터 버전, 하이퍼파라미터,
  평가 코드 실행 로그, 상세 지표 테이블.
- 보고서 내 모든 수치·차트는 [1]에서 실제 산출된 값만 사용한다.
- 사내 템플릿이 있으면 우선 적용하고, 없으면 기본 템플릿을 쓰되 템플릿 부재 사실을
  사용자에게 알린다.
- 보고서 초안 완성 후 **Human Checkpoint(보고서 최종 승인)**를 요청한다. 승인 전에는
  고객에게 전달하지 않는다.

## 산출물 체크리스트

- [ ] 성능 평가 결과 (지표 + baseline 대비 비교, 실행 로그 포함)
- [ ] 결과 분석 및 개선 포인트 요약 (가설/액션 아이템 구분)
- [ ] 고객向 보고서 (PPT)
- [ ] 내부용 상세 문서 (Markdown)

## 지켜야 할 규칙 (요약)

- 모든 산출물은 사람 승인 전까지 "draft"로 표시하고 고객에게 전달하지 않는다.
- 평가 결과를 임의로 수정/왜곡하지 않는다.
- 모든 결론에는 근거(지표 수치, baseline 비교)를 함께 제시한다.
- 성능/품질을 과장된 정성적 표현으로 서술하지 않는다(baseline 대비 수치로 표현).
- 평가가 불완전하거나 데이터/모델이 미비한 경우, 값을 임의로 채우지 않고
  "미완료/불확실" 상태와 원인을 명시한다.

## 참고
- 이 skill은 `data-model-agent` skill의 산출물을 입력으로 사용한다. 모델/데이터
  버전이 명시되지 않은 상태로 이 skill이 호출되면, 먼저 버전을 확인하는 것을
  최우선으로 한다.
