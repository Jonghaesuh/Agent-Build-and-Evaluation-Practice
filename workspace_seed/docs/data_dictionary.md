# 데이터 사전: `customers.csv`

- 생성일: 2026-07-09 (에이전트 파이프라인 검증용 합성 데이터)
- 행 수: 502 (의도적 중복 2건 포함)
- 대상 PRD: `sample_prd.md` (구독 서비스 고객 이탈 예측 시스템)

> 이 문서는 에이전트가 데이터 프로파일링 단계에서 스스로 발견해야 할 내용을
> 검증자가 미리 확인할 수 있도록 만든 **정답지 성격의 참고 문서**입니다.
> 에이전트에게 이 문서를 그대로 입력하지 말고, `customers.csv`만 준 뒤
> 에이전트의 프로파일링 결과와 이 문서를 비교하는 용도로 사용하세요.

## 컬럼 정의

| 컬럼명 | 타입 | 설명 | 결측/이상치 |
|---|---|---|---|
| `customer_id` | string | 고객 고유 ID | 2건 중복 존재 (데이터 파이프라인 오류 시뮬레이션) |
| `signup_days_ago` | int | 가입 후 경과 일수 | 없음 |
| `plan_type` | category | 가입 플랜 (`basic`/`standard`/`premium`) | `trial_promo` 값 8건 존재 — **학습 데이터 정의 시점에 없던 신규 카테고리** (PRD 3.3 참조) |
| `monthly_payment` | float | 월 결제 금액 | 약 2% 음수값(환불 오류 추정), 약 1% 극단적 고값(정상 범위의 6배 이상) |
| `last_login_days_ago` | float | 마지막 로그인 이후 경과 일수 | 약 5% 결측 |
| `login_count_90d` | int | 최근 90일 로그인 횟수 | 없음 (단, 가입 30일 미만 신규 고객은 값이 비정상적으로 낮게 나타남 — 데이터 부족이 아니라 실제 이용 기간이 짧은 것) |
| `support_tickets_90d` | float | 최근 90일 고객센터 문의 건수 | 약 12%(72건) 결측 — **문의 이력이 없는 경우 시스템이 NULL로 기록** (0건과는 다른 의미이므로 단순 0 대체 시 근거를 명시할 것) |
| `churned_next_30d` | int (0/1) | 이후 30일 내 실제 해지 여부 (학습/평가용 타깃) | 클래스 불균형 존재 — 이탈(1) 비율 약 5% |

## 알려진 데이터 이슈 (에이전트가 발견해야 할 포인트)

1. **중복 레코드**: `customer_id` 기준 중복 2건 → 전처리 설계 시 dedup 여부와 기준(최신 레코드 유지 등)을 명시해야 함
2. **신규 카테고리**: `trial_promo`는 PRD 3.3의 "신규 플랜이 추가될 수 있으므로 학습 시점에 없던 플랜 값 처리 방안 필요"에 해당하는 케이스. 단순 원-핫 인코딩 시 학습/추론 간 컬럼 불일치가 발생할 수 있음 → Fallback 로직 필요
3. **결제 이상치**: 음수값(환불 오류 추정)과 극단적 고값이 혼재. 단순 제거보다 원인 구분이 필요할 수 있음 (PRD 3.3 "결제 이력 이상치 예외 처리")
4. **의미가 다른 결측**: `support_tickets_90d`의 NULL은 "문의 없음"을 의미할 가능성이 높음. `last_login_days_ago`의 NULL은 원인이 불명확(로그 수집 실패 등)하므로 같은 방식으로 처리하면 안 됨
5. **신규 가입자 데이터 부족**: 가입 30일 미만 고객은 `login_count_90d` 등 지표가 구조적으로 낮게 나타남 → 이탈 위험군으로 잘못 분류되지 않도록 별도 처리 필요 (PRD 3.1)
6. **클래스 불균형**: 타깃 `churned_next_30d`의 양성 비율이 약 5%로 낮음 → 모델/평가 지표 선택 시 반드시 고려해야 함 (Accuracy 단독 사용 지양)
7. **개인정보 미포함**: 이름/연락처 등 직접 식별 정보는 포함하지 않음 (PRD 3.4 준수 확인용)
